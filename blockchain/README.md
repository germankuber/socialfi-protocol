# Blockchain

Polkadot SDK parachain, built with FRAME + Cumulus, runs on
`polkadot-omni-node`.

## Layout

| Path | Role |
| --- | --- |
| [`pallets/`](pallets/) | SocialFi FRAME pallets |
| [`primitives/social-notifications/`](primitives/social-notifications/) | `no_std` crate shared by pallets that emit Statement Store notifications |
| [`runtime/`](runtime/) | Parachain runtime — `construct_runtime!`, tx-extension pipeline, pallet configs |
| [`Dockerfile`](Dockerfile) | Image that packages a pre-built chain spec |
| [`zombienet.toml`](zombienet.toml) | Relay-backed local topology |
| `chain_spec.json` | Generated by the dev scripts; not tracked |

Runtime internals:

| File | Purpose |
| --- | --- |
| `runtime/src/lib.rs` | `construct_runtime!`, runtime APIs, opaque types |
| `runtime/src/configs/mod.rs` | Every pallet's `Config` impl + `NotificationStatementSubmitter` adapter |
| `runtime/src/configs/xcm_config.rs` | XCM configuration |
| `runtime/src/genesis_config_presets.rs` | Dev / testnet genesis presets |
| `runtime/src/weights/` | Benchmark-generated weights |

## Build & test

Runs from repo root. See [`../docs/INSTALL.md`](../docs/INSTALL.md)
for toolchain setup.

```bash
cargo build --release                        # runtime + all pallets
cargo test  --workspace                      # unit + mock-runtime tests
cargo test  --workspace --features runtime-benchmarks
```

WASM blob output: `target/release/wbuild/stack-template-runtime/stack_template_runtime.compact.compressed.wasm`.

## Running

Use the root `Makefile`:

- `make node` — solo-node dev loop.
- `make node-quiet` — same, consensus logs muted.

Under `polkadot-sdk stable2512-3` the solo-node path does **not**
expose Statement Store RPCs. Run a relay-backed topology (e.g. a
Zombienet script) when you need those locally.
